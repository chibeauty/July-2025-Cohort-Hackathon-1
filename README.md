# Gloria Ngwu - Personal Portfolio Website

Welcome! This is the source code for my personal portfolio website, designed for hackathon excellence. It showcases my journey, skills, and projects as a frontend developer and software engineering student.

---

## 🌟 Project Overview

This portfolio is a modern, fully responsive, and accessible website built with:

- **HTML5** for semantic, structured content
- **CSS3** for all styling, layout, and creative effects (no JavaScript, libraries, or frameworks)
- Custom gradients, card layouts, and subtle animations for a beautiful user experience
- Sections: Hero, About, Skills, Education, Interests, Projects, Contact

---

## 🎯 Purpose

- Present my background, technical skills, and projects
- Provide an easy way for recruiters or collaborators to contact me
- Demonstrate best practices in modern frontend development using only HTML and CSS

---

## 🧠 Features

- 100% raw HTML and CSS (no JS, libraries, or frameworks)
- Fully responsive (mobile-first) and accessible
- Sticky navigation bar with smooth section scrolling
- Projects grid (static, pure HTML/CSS)
- Downloadable CV
- Contact form (static, accessible)
- Creative CSS: gradients, card effects, subtle animations, and beautiful section styling

> **Note:** All interactive features are implemented using CSS only. Some features (like project filtering, modal popups, and light/dark mode) are not available to comply with competition rules.

## 🤝 Development Process & Collaboration

This project was developed as a solo hackathon submission. All code is original. The process included:

- Planning and wireframing the layout
- Writing semantic HTML and accessible markup
- Designing and refining the UI with CSS
- Iterative testing for responsiveness and accessibility
- Version control with frequent, descriptive commits

For feedback or collaboration, please open an issue or contact me via the portfolio contact form.

## 🧾 Code Commenting Guidelines

You already wrote clean HTML and CSS. Here’s how to add meaningful comments:

### HTML

Add comments to indicate the purpose of each section:

```html
<!-- ===== HEADER WITH NAVIGATION ===== -->
<header role="banner">
    ...
</header>

<!-- ===== HERO SECTION: Introduction ===== -->
<section class="hero" aria-labelledby="hero-heading">
    ...
</section>

<!-- ===== ABOUT SECTION ===== -->
<section id="about" aria-labelledby="about-heading">
    ...
</section>
```

### CSS

Use comment headers to break down styles into logical sections:

```css
/* -------------------------
   GLOBAL RESET & BASE STYLES
---------------------------- */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* -------------------------
   HEADER STYLES
---------------------------- */
header {
  background: #2563eb;
  ...
}

/* -------------------------
   HERO SECTION STYLES
---------------------------- */
.hero {
  background: linear-gradient(...);
  ...
}
```
